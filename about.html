CRUD Implementation Overview
  • Create: create.html → adds new records to local storage.
  • Read: read.html → searchable data table view.
  • Update: update.html → choose or load state via ?id=XX.
  • Delete: delete.html → confirmation-based removal.
MVC Architecture
  • Model: assets/model.js — persistent localStorage service (v3 seed with 50 states).
  • Controller: assets/controller.js — handles CRUD actions & data binding.
  • View: HTML pages with modular scripts and charts.js visualizations.
Notable Technical Challenges & Solutions
  • KPI Blank Fix: Added safe guards to update KPIs even when charts not on page.
  • Update UX: Added dynamic state chooser if no query parameter is provided.
  • External Insights: Implemented retry/error handling for DataUSA API.
  • Persistence: Used singleton localStorage model with schema versioning.
