About this Project

The US States Dashboard is a web application we built to learn how to turn a basic website into a fully functional app using the MVC (Model-View-Controller) structure. It allows users to add, edit, view, and delete data about U.S. states while keeping the information saved using localStorage. We also connected it with the World Bank API to show live data like GDP and population. The app includes charts made with Chart.js for clear data visualization and a chatbot for user interaction. Everything is deployed on Microsoft Azure, making it a professional and accessible web project that demonstrates real-world CRUD functionality.

API Endpoints Used (Read-only)

We used the World Bank Open Data API to fetch real-time information about the United States. These APIs help us display up-to-date insights such as GDP and population on the homepage without manually entering data. This makes our application more dynamic and realistic by including live external data.

  • GDP (current US$): https://api.worldbank.org/v2/country/USA/indicator/NY.GDP.MKTP.CD?format=json&per_page=8&date=2019:2025
  • Population (total): https://api.worldbank.org/v2/country/USA/indicator/SP.POP.TOTL?format=json&per_page=8&date=2019:2025

Overview of CRUD Implementation

Our project is based on the MVC pattern, and we implemented full CRUD functionality (Create, Read, Update, Delete) using JavaScript and localStorage for data persistence. Each page of the app performs a specific part of the CRUD process, and all data changes are synchronized automatically across the dashboard. This helped us understand how frontend logic and data storage work together in a real application.

  • Create: Users can add new states using the form on create.html. The data is saved in localStorage through the api.upsert() function.
  • Read: The list of states is shown on data.html, which loads all saved data from localStorage. The homepage also shows summarized data and charts.
  • Update: The update.html page lets users edit any existing state’s details and saves the new values back into localStorage.
  • Delete: The delete.html page allows users to remove a state permanently using the api.remove() function.

Notable Technical Challenges & Solutions

  • Consistent Centering Across Pages: Introduced .panel-inner with max-width and global centering overrides to prevent left-skew from flex parents.
  • External Insights Reliability: Fetched recent year ranges from World Bank and handled missing data gracefully, ensuring the UI never breaks.
  • Single Source of Truth: All UI components use one shared api service built on top of localStorage for reliable data updates.
  • Charts Separation: Moved all charts to charts.html to keep the homepage clean and improve loading speed.
