Engineering Notes
Overview of CRUD Implementation

The application follows a simple MVC pattern in the browser. The model persists state data using localStorage with a small data-access layer. Controller functions validate and transform input before writing to storage. Views are organized as separate pages: Create adds new states with client-side validation, Data Table lists all states with inline action links, Update loads and edits a selected state, and Delete removes a selection. The charts page reads from the same data source to keep visuals consistent.

Notable Technical Challenges & Solutions
  • Empty dataset on first load. Charts and KPIs now handle an empty store by showing a clear notice and using a small demo dataset until the user adds records.
  • Number formatting and overflow. Long values are formatted with thousands separators and abbreviated where needed to avoid overflow in KPI and chart labels.
  • Case-sensitive assets on GitHub Pages. Image paths were corrected to match file casing exactly (e.g., hongxu.JPG vs hongxu.jpg).
  • Consistent theming. Reused a shared navbar and card layout so typography and colors read well across pages.
