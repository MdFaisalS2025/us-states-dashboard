CRUD Implementation
  • Create: create.html → DB.createState
  • Read: read.html (searchable table)
  • Update: update.html with chooser or ?id=XX
  • Delete: delete.html with confirmation
MVC Architecture
  • Model: assets/model.js (localStorage, versioned seed)
  • Controller: assets/controller.js
  • View: HTML pages + assets/charts.js
Notable Challenges & Solutions
  • KPI blank → render guards + setKPIs runs first.
  • Update UX → added dropdown loader when no ?id.
  • External API reliability → switched to World Bank (CORS OK) with error UI.
  • Persistence → singleton localStorage with schema versioning.
