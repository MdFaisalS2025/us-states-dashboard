Data Model (ERD)

Overview of CRUD Implementation

The application follows a lightweight MVC structure. Model logic in assets/model.js manages persistent state in localStorage (key us_states_v1), exposing getAll/create/update/remove and aggregate totals(). Controller code in assets/controller.js binds form events, renders tables, updates views, and injects a shared navbar/footer for consistent layout. Views are the HTML pages. Charts are rendered with Chart.js in assets/charts.js.

When a record is created/updated/deleted, the table, KPIs, and charts reflect the change immediately because all pages read from the same persisted model. External Insights use the World Bank API for GDP and population (read-only) with a local fallback if the request is blocked.

Notable Technical Challenges and Solutions

  • Empty dataset during grading: Seeded a realistic initial dataset in model.js so the dashboard is never blank and still behaves like a CRUD app through local persistence.
  • Cross-origin blocks on APIs: Wrapped the World Bank calls with a try/catch and added a local fallback snapshot to keep the homepage informative when network policies block requests.
  • Design consistency: Centralized theme tokens and shared chrome injection to guarantee identical navigation, typography, colors, and spacing across all pages.
  • Large ERD image: Contained and scaled with a responsive wrapper to avoid layout overflow.
