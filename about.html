Data Model (ERD)
Overview of CRUD Implementation
  • Model: LocalStorage wrapper (assets/model.js) with seed data and helpers for add, update, remove, list.
  • View: Plain HTML templates styled with assets/styles.css. Reusable cards/tables.
  • Controller: assets/controller.js wires forms, table actions, totals, and page routing.
  • Charts: assets/charts.js renders donut/bar/line using Chart.js from the current dataset.
  • API: assets/api.js fetches World Bank GDP & Population for the Home page.
Notable Technical Challenges & Solutions
  • Inconsistent layout & unreadable text: Unified a dark theme with higher contrast and consistent spacing.
  • Charts not loading: Ensured Chart.js is loaded on data.html and corrected canvas IDs & dataset guards.
  • Empty dataset issues: Added safe seeding and empty-state handling so pages don’t break.
  • Image loading on GitHub Pages: Fixed case-sensitive filenames (faisal.JPG, hongxu.JPG, jonathan.jpg) and constrained sizes via CSS.
  • Duplicate bot widget: Kept a single Botpress embed on every page.
